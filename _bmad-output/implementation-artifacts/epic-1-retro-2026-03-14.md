## Challenges identifies

### Pattern recurrent 1 — Documentation architecture decalee (Tailwind v3 vs v4)
L'architecture originale referencait des patterns Tailwind v3 (`tailwind.config.mjs`, `@tailwind directives`). Le projet utilise Tailwind v4 (`@tailwindcss/vite`, `@theme` en CSS). Corrige en Story 1.1 mais mentionne comme "ALERTE CRITIQUE" dans chaque story suivante.

### Pattern recurrent 2 — i18n et valeurs en dur
3 stories sur 5 ont eu des corrections liees a des textes ou valeurs non internationalises (labels navigation, noms de configs, format de prix).

### Pattern recurrent 3 — Accessibilite amelioree en review
Chaque code review a releve des ameliorations d'accessibilite (aria-labels manquants, semantique HTML incorrecte, prefixes `focus:` oublies). Le pattern s'ameliore au fil des stories.

### Images existantes ignorees
5 fichiers images existaient dans `/images/` a la racine du projet (`salon.png`, `chambre1.png`, `chambre2.png`, `chambre3.png`, `logo.png`). Le dev a cree des placeholders (divs colores) sans verifier les assets existants.

### URLs Airbnb restees en placeholder
Les vrais liens Airbnb existaient mais n'ont pas ete fournis pendant le developpement.

### Deploiement non effectue
Pas de remote Git configure pendant le developpement. Story 1.5 non commitee. Deploiement Vercel realise en fin d'epic seulement.

### Redirection racine defaillante
La page `/` sur Vercel ne redirige pas automatiquement vers `/fr/` — le visiteur doit cliquer. Pas de detection de langue du navigateur.

## Decouverte significative — Changement de direction UX

### Homepage simplifiee : Option A (Immersif pur)

La homepage actuelle (8 sections hybride Immersif-Corporate) est jugee trop chargee par Onizuka. Decision prise de passer a l'**Option A** :

**Avant (8 sections) :**
Hero → TrustBar → ApartmentGrid → KwaboSection → QuartierTeaser → HostSection → FinalCta → Footer

**Apres (4 sections) :**
Hero → ApartmentGrid → CTA Final → Footer

Les composants retires (TrustBar, KwaboSection, QuartierTeaser, HostSection) restent dans le code pour reutilisation dans les pages dediees de l'Epic 2.

Un fichier de comparaison visuelle a ete cree : `_bmad-output/homepage-options.html`

### Impact sur Epic 2
- Les composants Kwabo, Quartier, Host migrent vers leurs pages dediees respectives
- Le plan de l'Epic 2 reste valide — les pages Appartement, Fidjrosse, Votre Hote sont toujours prevues
- Les composants existants accelereront le developpement de l'Epic 2
